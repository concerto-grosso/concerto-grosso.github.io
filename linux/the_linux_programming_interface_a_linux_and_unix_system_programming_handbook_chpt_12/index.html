《Linux/UNIX系统编程手册》第12章 系统和进程信息

发表于 2024-12-18 更新于 2025-01-16 分类于 Linux 本文字数： 4.3k

　　对于早期UNIX实现，没有简单的方法获取系统和进程信息。部分早期UNIX实现通过允许特权进程访问内核内存中的数据结构来解决这个问题，但这个方法也有很多问题 (该方法需要内核数据结构的专业知识，并且这些数据结构可能因为内核版本而异)。

　　为了提供更简单的访问内核信息的方法，很多现代UNIX实现都提供了/proc虚拟文件系统 (SUSv3没有说明该文件系统)。/proc文件系统位于目录/proc下，其中包含各种公开内核信息的文件，在某些情况下，进程可以通过常规文件I/O系统调用来修改这些文件。

　　/proc文件系统中的文件和目录并非位于磁盘，而是在进程访问它们时内核动态创建。因此，/proc文件系统被称为虚拟文件系统。

12.1 /proc文件系统

12.1.1 /proc下的进程信息

　　对于系统中的每个进程，内核都提供了对应的名为/proc/PID的目录，其中PID就是进程的进程ID。该目录下是包含进程相关信息的各种文件和子目录。

文件描述 (进程属性)
cmdline以字符\0分隔的命令行参数
cwd指向当前工作目录的符号链接
environ以字符\0分隔的形如NAME=value的环境变量
exe指向正在执行的文件的符号链接
fd包含指向该进程打开的文件的符号链接的目录
maps内存映射
mem进程虚拟内存 (在执行I/O前必须先通过lseek()来将文件偏移量修改为有效值)
mounts该进程的挂载点
root指向根目录的符号链接
status各种信息 (例如，进程ID、凭证、内存使用情况和信号)
task对于进程的每个线程，该目录下都有1个对应的子目录 (Linux 2.6及以上版本)

　　上表列出了部分所有/proc/PID目录都包含的文件。

　　● 对于进程打开的每个文件描述符，目录/proc/PID/fd下都有1个指向该文件描述符的符号链接。这些符号链接都有1个与描述符号对应的名称 (例如，/proc/1968/1是指向进程1968的标准输出的符号链接)。此外，每个进程都可以通过符号链接/proc/self来访问自己的/proc/PID目录。

　　● Linux 2.4添加了线程组的概念以支持POSIX线程模型。线程组中的线程的某些属性是不同的，所以Linux 2.4在/proc/PID目录下添加了子目录task。内核为进程的每个线程都提供了名为/proc/PID/task/TID的子目录 (其中TID就是线程的线程ID)，该子目录下的文件和子目录与/proc/PID下的相同。对于同一进程的线程，该目录下的文件中的很多信息是相同的，因为这些线程共享很多属性。

12.1.2 /proc下的系统信息

　　/proc下的文件和子目录提供了对系统信息的访问。

目录该目录下的文件公开的信息

/proc/net关于网络和套接字的状态信息

12.1.3 访问/proc文件

　　/proc下的文件经常通过shell脚本访问 (大多数包含多个值的/proc文件都可以使用Python或Perl等脚本语言轻松解析)。

　　/proc文件也能通过使用常规文件I/O系统调用的程序访问，访问这些文件时存在以下限制：

　　● 有些/proc文件 (例如，/proc/PID目录下的大部分文件) 是只读的，所以这些文件只能用于展示内核信息。

　　● 有些/proc文件 (例如，/proc/PID下的部分文件) 只能由文件所有者或特权进程读取。

　　● 除了/proc/PID目录下的文件之外，大部分其他/proc文件的所有者都是root，其中可修改的文件只能由root修改。

　　/proc/PID目录是不稳定的，这些目录只会在对应的进程创建时出现，对应的进程终止时消失。因此，当访问/proc/PID目录下的内容时，需要先确定对应的进程是否已终止以及该目录是否已被删除。

12.2 系统ID

　　系统调用uname()用于获取关于运行应用程序的主机系统的标识信息，结果存储在参数utsbuf指向的结构体中。

　　结构体utsname中的成员sysname、release、version和machine由内核自动设置。成员nodename返回通过系统调用sethostname()设置的值，该值有时与系统DNS域名中的前缀名相似。成员domainname返回通过系统调用setdomainname()设置的值，这是主机的网络信息服务域名 (与主机的DNS域名无关)。

　　系统调用sethostname()和setdomainname()很少用于应用程序。通常，主机名和NIS域名在引导时由启动脚本设置。

　　系统调用gethostname()与sethostname()相反，用于检索系统主机名。系统主机名也可以通过命令hostname和特定Linux的文件/proc/hostname来查看和修改。

　　系统调用getdomainname()与setdomainname()相反，用于检索NIS域名。NIS域名也可以通过命令domainname和特定Linux的文件/proc/domainname来查看和修改。

　　对于Linux，目录/proc/sys/kernel下的只读文件ostype、osrelease和version提供的信息与通过成员sysname、release和version返回的信息相同。文件/proc/version也包含这些信息，并且还包含关于内核编译步骤的信息 (即执行编译的用户和主机的名称以及使用的gcc版本)。

　　SUSv3详细说明了uname()，但没有说明结构体utsname中的成员的长度，仅要求它们以字符\0结尾。对于Linux，这些成员的长度都是65字节 (包括结尾的字符\0)。其他UNIX实现则不尽相同。
