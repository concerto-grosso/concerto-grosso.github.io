《Linux/UNIX系统编程手册》第46章 System V消息队列

发表于 2026-03-21 更新于 2026-03-22 分类于 Linux 本文字数： 8.3k

　　系统调用msgsnd()和msgrcv()会对消息队列执行I/O。这2个系统调用的第1个参数都是消息队列标识符。第2个参数指向程序员定义的结构体的指针，该结构体用于存储正在发送或接收的消息，其形式通常如下：

　　这个定义实际上只是简写，用于表示消息的第1部分是类型 (指定为长整数)，而消息的剩余部分是由程序员定义的任意长度和内容 (不一定是字符串) 的结构体 (所以msgsnd()和msgrcv()的第2个参数的类型为void *)。消息体的长度可以为0，这在某些情况下很有用 (例如，传递的消息可以仅以消息类型进行编码)。

　　所有消息队列都有1个关联的结构体msqid_ds (该名称中的缩写msq是消息队列接口中唯一采用这种缩写的情况)。

　　成员msg_qnum和msg_qbytes的数据类型msgqnum_t和msglen_t是无符号整数类型，并且SUSv3也详细说明了这些数据类型。

　　各种消息队列系统调用都会隐式地更新结构体msqid_ds的成员。系统调用msgctl()的操作IPC_SET可用于显式地修改该结构体的部分成员。

　　● 成员msg_perm会在创建消息队列时被初始化 (见45.3)，其子成员uid、gid和mode可以通过操作IPC_SET更新。

　　● 成员msg_stime和msg_rtime会在创建消息队列时被设置为0。后续每次成功调用msgsnd()或msgrcv()会将msg_stime或msg_rtime更新为当前时间 (自纪元以来的秒数)。

　　● 成员msg_ctime会在创建消息队列以及后续执行操作IPC_SET时被设置为当前时间。

　　● 成员__msg_cbytes会在创建消息队列时被设置为0。后续每次成功调用msgsnd()或msgrcv()会将该成员更新为消息队列中所有消息的消息体的字节数之和。

　　● 成员msg_qnum会在创建消息队列时被设置为0，其值会随着每次成功调用msgsnd()或msgrcv()而增减。

　　● 成员msg_qbytes定义了消息队列中所有消息的消息体的字节数之和的上限，它会在创建消息队列时被初始化为限制MSGMNB。特权进程 (能力CAP_SYS_RESOURCE) 可以通过操作IPC_SET将其值设置为0到INT_MAX之间的值，而非特权进程只能将其值设置为0到MSGMNB之间的值。

　　● 成员msg_lspid和msg_lrpid会在创建消息队列时被设置为0。后续每次成功调用msgsnd()或msgrcv()会将msg_lspid或msg_lrpid设置为调用进程的进程ID。

　　SUSv3说明了以上所有成员，除了__msg_cbytes。尽管如此，大多数UNIX实现都提供了等同于__msg_cbytes的成员。
