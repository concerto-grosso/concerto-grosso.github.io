《Linux/UNIX系统编程手册》第52章 POSIX消息队列

发表于 2026-05-26 更新于 2026-05-28 分类于 Linux 本文字数： 7.6k

　　POSIX消息队列与System V消息队列类似。但是，两者存在一些显著的差异：

　　● POSIX消息队列会被引用计数。被标记为待删除的消息队列只会在所有进程都将其关闭后才会被删除。

　　● System V消息有类型字段，并且通过msgrcv()接收信息时可以进行筛选。相比之下，POSIX消息有关联的优先级，并且总是会严格按优先级排序。

　　● POSIX消息队列允许在空队列中有消息可用时异步通知进程。

　　POSIX消息队列所需的支持在Linux 2.6.6中添加 (此外，还需要glibc 2.3.4或更高版本)。POSIX消息队列支持是可选内核组件，可以通过选项CONFIG_POSIX_MQUEUE配置。

　　POSIX消息队列API的主要函数如下：

　　● 函数mq_open()会创建消息队列或打开消息队列，并返回消息队列描述符供后续调用使用。

　　● 函数mq_send()会向队列写入1条消息。

　　● 函数mq_receive()会从队列读取1条消息。

　　● 函数mq_close()会关闭进程之前打开的消息队列。

　　● 函数mq_unlink()会删除消息队列名并将消息队列标记为待删除。

　　此外，POSIX消息队列API还包含以下特性：

　　● 每个消息队列有多个关联属性。部分属性会在通过mq_open()创建或打开队列时被设置。函数mq_getatrr()和mq_setattr()可用于检索和修改队列属性。

　　● 函数mq_notify()允许进程注册以接收来自队列的消息通知。当进程注册后，进程通过接收信号或由独立线程调用函数来获知消息可用。

　　消息队列描述符与打开的消息队列之间的关系类似于文件描述符与打开的文件之间的关系。消息队列描述符是进程级句柄，它引用系统级的打开的消息队列描述表中的某个条目，而该条目又引用某个消息队列对象。

　　对于Linux，POSIX消息队列被实现为虚拟文件系统中的i-node，而消息队列描述符和打开的消息队列描述分别被实现为文件描述符和打开的文件 (描述)。但是，SUSv3没有说明这一点，并且在某些UNIX实现中也不适用。

　　上图有助于阐明消息队列描述符的使用细节 (类似于文件描述符的使用)：

　　● 打开的消息队列描述有多个关联的标志。SUSv3仅详细说明了标志O_NONBLOCK (决定I/O是否以阻塞方式进行)。

　　● 多个进程可以持有引用同一消息队列描述的消息队列描述符 (进程打开消息队列并调用fork())。这些描述符共享标志O_NONBLOCK的状态。

　　● 多个进程可以持有引用不同消息队列描述的消息队列描述符，而这些消息队列描述又引用同一消息队列 (不同进程各自调用mq_open()打开消息队列)。
