《Linux/UNIX系统编程手册》第12章 系统和进程信息

发表于 2024-12-18 更新于 2025-01-16 分类于 Linux 本文字数： 4.3k

　　对于早期UNIX实现，没有简单的方法获取系统和进程信息。部分早期UNIX实现通过允许特权进程访问内核内存中的数据结构来解决这个问题，但这个方法也有很多问题 (该方法需要内核数据结构的专业知识，并且这些数据结构可能因为内核版本而异)。

　　为了提供更简单的访问内核信息的方法，很多现代UNIX实现都提供了/proc虚拟文件系统 (SUSv3没有说明该文件系统)。/proc文件系统位于目录/proc下，其中包含各种公开内核信息的文件，在某些情况下，进程可以通过常规文件I/O系统调用来修改这些文件。

　　/proc文件系统中的文件和目录并非位于磁盘，而是在进程访问它们时内核动态创建。因此，/proc文件系统被称为虚拟文件系统。

12.1 /proc文件系统

12.1.1 /proc下的进程信息

　　对于系统中的每个进程，内核都提供了对应的名为/proc/PID的目录，其中PID就是进程的进程ID。该目录下是包含进程相关信息的各种文件和子目录。

文件描述 (进程属性)
cmdline以字符\0分隔的命令行参数
cwd指向当前工作目录的符号链接
environ以字符\0分隔的形如NAME=value的环境变量
exe指向正在执行的文件的符号链接
fd包含指向该进程打开的文件的符号链接的目录
maps内存映射
mem进程虚拟内存 (在执行I/O前必须先通过lseek()来将文件偏移量修改为有效值)
mounts该进程的挂载点
root指向根目录的符号链接
status各种信息 (例如，进程ID、凭证、内存使用情况和信号)
task对于进程的每个线程，该目录下都有1个对应的子目录 (Linux 2.6及以上版本)

　　上表列出了部分所有/proc/PID目录都包含的文件。

　　● 对于进程打开的每个文件描述符，目录/proc/PID/fd下都有1个指向该文件描述符的符号链接。这些符号链接都有1个与描述符号对应的名称 (例如，/proc/1968/1是指向进程1968的标准输出的符号链接)。此外，每个进程都可以通过符号链接/proc/self来访问自己的/proc/PID目录。

　　● Linux 2.4添加了线程组的概念以支持POSIX线程模型。线程组中的线程的某些属性是不同的，所以Linux 2.4在/proc/PID目录下添加了子目录task。内核为进程的每个线程都提供了名为/proc/PID/task/TID的子目录 (其中TID就是线程的线程ID)，该子目录下的文件和子目录与/proc/PID下的相同。对于同一进程的线程，该目录下的文件中的很多信息是相同的，因为这些线程共享很多属性。
